# BLACKBOX.ROCKS |ψ⟩ Q — Public feedback tracker

Public issue tracker, feature requests, and discussion forum for
**[BLACKBOX.ROCKS](https://blackbox.rocks) |ψ⟩ Q — Quantum Full Platform Optimizer**,
the multi-tenant SaaS for portfolio optimisation across quantum and classical
compute backends.

> The product source code is **not** in this repository (it lives in a private
> engineering repo to protect the IP). This repository exists so that
> customers, evaluators, and the wider quantum-computing community can engage
> with the product in the open.

## Where to start

| What you want to do | How |
|---|---|
| **Report a bug** | [Open a Bug Report](https://github.com/lxtececo/blackbox-rocks-q-feedback/issues/new?template=bug.yml) |
| **Request a feature** | [Open a Feature Request](https://github.com/lxtececo/blackbox-rocks-q-feedback/issues/new?template=feature.yml) |
| **Propose a quantum method / provider** | [Open a Quantum Idea](https://github.com/lxtececo/blackbox-rocks-q-feedback/issues/new?template=quantum_idea.yml) |
| **Ask a question** | [Open a Question](https://github.com/lxtececo/blackbox-rocks-q-feedback/issues/new?template=question.yml) or [start a Discussion](https://github.com/lxtececo/blackbox-rocks-q-feedback/discussions) |
| **Anything sensitive / commercial** | Email [feedback@blackbox.rocks](mailto:feedback@blackbox.rocks) |
| **Security disclosure** | Email [security@blackbox.rocks](mailto:security@blackbox.rocks) — **do not** file publicly |

## In-app alternative

Every page of the app at
[`quantumfpo.staging.blackbox.rocks`](https://quantumfpo.staging.blackbox.rocks)
and (soon) `app.quantumfpo.com` has a **Send feedback** button in the footer.
It posts directly into our triage queue — no GitHub account needed.

## Triage SLA

We aim to:

- Acknowledge new issues within **2 business days**.
- Triage and label within **5 business days**.
- Reflect accepted items on the public roadmap within the next 4-week demo

complex product. Quality input gets quality follow-up.
